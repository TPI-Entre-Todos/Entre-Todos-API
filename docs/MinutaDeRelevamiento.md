# Proceso EntreTodos

## Contexto

Actualmente, los viajes grupales entre amigos, familiares o compañeros implican la realización de múltiples gastos compartidos, tales como alojamiento, transporte, comidas y actividades recreativas.

En la mayoría de los casos, estos gastos son abonados por distintos participantes y luego se intenta dividir el total de manera equitativa. Sin embargo, este proceso suele realizarse de forma manual, utilizando anotaciones informales, calculadoras o aplicaciones de mensajería, lo que genera confusión, errores en los cálculos y dificultades al momento de saldar las deudas.

Por este motivo, surge la necesidad de contar con un sistema que permita organizar y gestionar los gastos compartidos de forma clara, automatizada y accesible para todos los participantes del viaje.

## Proceso con el sistema deseado

Cuando un usuario desea gestionar los gastos de un viaje grupal, accede al sistema mediante su usuario y contraseña.
Una vez dentro, el sistema muestra un panel principal donde el usuario puede visualizar sus viajes y seleccionar uno existente o crear un nuevo viaje.

Al crear un viaje, el usuario ingresa información básica como nombre, descripción y fechas. Automáticamente, el usuario creador queda asignado como organizador del viaje.

El organizador puede invitar a otros usuarios a participar del viaje. Los usuarios invitados pueden aceptar la invitación y acceder al viaje, quedando registrados como participantes.

Dentro del viaje, cualquier participante puede registrar un gasto. Para ello, el sistema solicita completar información como descripción del gasto, monto total, fecha, quién realizó el pago y qué participantes forman parte del mismo.

El usuario puede seleccionar el tipo de división del gasto. Puede optar por una división igualitaria entre todos los participantes o definir montos personalizados para cada uno.

Una vez registrado el gasto, el sistema calcula automáticamente cuánto corresponde a cada participante y actualiza los balances individuales dentro del viaje.

Los usuarios pueden consultar en todo momento su saldo, visualizando cuánto deben o cuánto les deben los demás participantes.
Cuando un usuario desea saldar una deuda, puede registrar un pago indicando el monto, el destinatario y la fecha. El sistema actualiza automáticamente los balances correspondientes.

El sistema también permite visualizar el estado general del viaje, indicando si existen deudas pendientes o si el mismo se encuentra completamente saldado.

Además, el sistema puede calcular una liquidación optimizada, determinando la menor cantidad de pagos necesarios para saldar todas las deudas entre los participantes.

Durante todo el proceso, los usuarios reciben notificaciones sobre nuevos gastos, pagos registrados o cambios relevantes dentro del viaje.

#### 🔹 **Notas importantes**

- Un usuario puede participar en múltiples viajes
- Un usuario puede ser organizador en un viaje y participante en otro
- Un gasto puede involucrar a uno o varios participantes
- Los pagos pueden ser parciales o totales
- Los balances se actualizan automáticamente ante cada operación
